## TL;DR

- Refly.AI 定位为「Vibe Workflow」产品，结合 Agent 的动态性与传统 Workflow 的可控性
- 核心创新：每个节点都是一个 Agent（配 2-3 个工具），而非传统 n8n 的单一 API 调用
- 一句话生成 Workflow，一个 Refly 节点 ≈ 20 个 n8n 节点
- 目标用户：自媒体创作者、内容生产者（非企业自动化场景）
- 长期愿景：通过收集用户行为数据（action），实现「predict next action」的个性化 AGI

---

## 背景与动机

当前 Workflow 产品存在两极问题：
- **Agent 端（如 Manus）**：成本高、稳定性差、结果不一致
- **传统 Workflow（n8n/Dify）**：复杂流程需写代码，对非程序员不友好

Refly.AI 希望在两者之间找到平衡点。

---

## 核心方案

### 产品架构

1. **Agent 白盒化**：提供 Agent Editor，用户只需写 prompt + 选 tools
2. **沙箱执行**：每个 Agent 配一台虚拟电脑，可写代码、调用 tools、操作文件
3. **分层模型调用**：
   - 强模型（GPT-5/Claude Sonnet 4.5）：负责任务拆解、工具选择
   - 轻量模型（Kimi K2）：负责执行简单任务，成本低、准确率高

### 成本对比

| 场景 | n8n | Refly.AI | Manus |
|------|-----|----------|-------|
| 搭建成本 | 3-6 小时 + 数十万 token | 一句话 + 几千 token | - |
| 单次执行 | - | 0.1-1 元 | 数美元 |

### 准确性取舍

- 放弃 100% 准确性，换取 10 倍门槛降低
- 聚焦内容生成场景（小红书文案、PPT、播客），而非企业自动化
- 目标：提供 80% 有用的结果，用户可二次编辑

---

## 数据飞轮

Refly.AI 真正收集的不是「知识」，而是用户的「行为数据」：

1. **Workflow 生成过程**：用户修改节点 prompt/tool = 给模型反馈
2. **多轮交互**：从模糊需求到最终结果的完整路径
3. **运行成功**：强正反馈信号
4. **社区分享**：其他用户的评分、点赞

> "用户的行为数据才是真正的数据飞轮"

长期目标：基于行为数据做「predict next action」，实现一句话端到端完成任务。

## 产品演进

1. **剪藏插件** → data connection 难规模化
2. **DeepResearch** → context 管理门槛高
3. **画布形态** → 爆火但难 scale，专业用户为主
4. **Vibe Workflow** → 当前形态，降复杂度 + 放大泛化性

关键转折点（2024 年 8 月）：
- 认识到画布形态「解释不清楚」，不是能 scale 的好形态
- 决定回归 Workflow，但用 Agent 重构每个节点

---

## 竞争壁垒

1. **工程复杂度**：三层架构（Agent 生成 Workflow → Agent 节点 → Tools 体系）
2. **团队认知**：「可能是地球上最懂 AI + Workflow 的团队」
3. **模型边界利用**：站在模型肩膀上，模型越强/越便宜，产品越强/越便宜
4. **窗口期**：小公司 3-6 个月难追上，大厂需 50-100 人团队干 1-2 年
